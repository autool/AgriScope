## 本轮最新增量

### 质量判定影像证据

- 全量质检批次固化实际业务影像编号、采集时间、受控 URI、大小、分辨率、SHA-256 及规范化摘要。
- 最新 `operational` 影像真正发生变化时，在影像入库事务内统一清理当前检查、关闭开放的自动规则问题、重开任务并保留历史质检账本。
- 自检提交重新核对影像摘要；旧批次被标记为 `legacy_unavailable`，不能继续作为提交证据。
- 前端展示影像证据和重检原因，成果包保存完整质检账本。

### 公开 Landsat 多景联合覆盖

- 公开检索结果使用 PostGIS 对真实 STAC Polygon/MultiPolygon 足迹计算逐景覆盖比例，不再用场景 bbox 冒充真实覆盖。
- 支持选择一景完整覆盖，或选择 2–10 景跨轨候选联合覆盖。
- 批次导入前由服务端重新获取所有 Item，并以 `ST_UnaryUnion`、`ST_Covers` 和 geography 面积执行权威联合覆盖门禁。
- 每景只裁取自身与共同目标范围的交集矩形；全部临时实体成功后才进入统一影像批次事务。
- GeoTIFF 保存查询范围、实际裁取范围、逐景覆盖率、联合覆盖率、景数和覆盖算法依据；SAS 仍不进入浏览器、数据库或文件标签。
- 当前前端使用 bbox 并集做快速选景引导，最终结论始终以后端真实 STAC 足迹为准；覆盖不足会在申请 SAS 和下载 COG 前拒绝。

## 当前验证证据

- 后端全量：392 项测试通过，Ruff 通过。
- 最新公开影像增量：8 项针对性测试通过，覆盖单景、跨轨联合覆盖、覆盖缺口阻断和失败清理。
- 前端：TypeScript 严格检查和 ESLint 通过。
- PostgreSQL/PostGIS：真实双景联合覆盖为 100%，单景不完整覆盖为 33.33%，空间分析事务只读并已回滚。
- 真实 Planetary Computer 检索：接口返回 20 景候选及 `STAC_GEOMETRY_POSTGIS_GEOGRAPHY` 覆盖依据；3°×3° 跨轨窗口没有单景完整覆盖时，能返回不同 WRS Path/Row 的部分覆盖候选。
- 真实覆盖阻断：单景只覆盖目标范围 38.16% 时，批次接口在签名和下载前返回 HTTP 400，没有创建资产或临时实体。
- 数据库迁移：质量影像快照迁移连续执行两次通过；带真实行政区和地块数据的空库初始化通过。

## 暂停点与后续优先级

1. 增加“所选 Item 真实联合覆盖预校验”接口或服务端推荐选景，使前端在正式导入前直接显示真实 Polygon 联合覆盖率；当前 bbox 引导可能高估倾斜轨道边缘，后端会正确阻断但仍可改善操作效率。
2. 按年度和轨道系统建设完整历史影像清单，完成跨轨镶嵌、年度覆盖缺口统计和省域分块生产；现有五景 Landsat-5 仍是可追溯子区语料，不代表 1980–2024 全省历史库。
3. 继续建设区域网平差、高级缝线/匀色和全省规模性能压测。
4. 使用甲方真实项目数据执行完整 UAT：影像入库 → 解译 → 全量质检 → 外业核查 → 三级审核 → 统计/报告 → 成果包/离线介质。
5. 补充飞控实时接入、设备大规模并发验证和涉密生产环境的安全测评。
